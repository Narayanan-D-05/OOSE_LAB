01 Overview

The Stock Maintenance System (SMS) is a web-based inventory management application that replaces manual stock registers and spreadsheet workflows. It provides real-time tracking of products, suppliers, stock levels, purchase orders, customer transactions, and invoice generation.

This PRD defines the complete product scope for building SMS as a live web application with Supabase as the database backend and Vercel for hosting. All users access the system through a browser — no native app required for the initial version.

This document translates the SRS (Ex No. 02) and UML diagrams (Ex No. 03, 05) into actionable engineering requirements with Supabase-specific data models, live deployment targets, and a phased delivery roadmap.

04 Feature Scope

🔒

User Authentication & RBAC

Supabase Auth with email/password login. Role-based access enforced via Row Level Security (RLS) policies on every table.

📦

Product Management

Full CRUD for products including name, SKU, category, unit of measure, price, and minimum stock threshold.

🏭

Supplier Management

Maintain supplier profiles, link suppliers to products, and associate suppliers with purchase orders.

📊

Stock Control (In & Out)

Transactional Stock In (procurement) and Stock Out (sales). Auto-calculates current stock on every transaction.

🛒

Create, send, and track POs. Supplier confirms delivery; system auto-triggers Stock In on approval.

👥

Customer Management

Maintain customer profiles, track purchase history, and associate customers with invoices and payments.

🧾

Auto-generate PDF invoices on confirmed sales. Invoice history per customer, with download and email dispatch.

📈

Reports & Analytics

Stock level report, transaction history, low-stock alerts, supplier performance, and sales summary dashboard.

🔔

Low Stock Alerts

Automated email notifications (via Supabase Edge Functions) when stock drops below configured threshold.

Key Database View

A Supabase view computes live stock quantities to avoid redundant storage:

-- View: current_stock
CREATE OR REPLACE VIEW current_stock AS
  SELECT
    p.id         AS product_id,
    p.name       AS product_name,
    p.sku,
    p.reorder_threshold,
    COALESCE(SUM(
      CASE WHEN st.type = 'STOCK_IN'  THEN st.quantity
           WHEN st.type = 'STOCK_OUT' THEN -st.quantity
      END
    ), 0) AS current_quantity
  FROM products p
  LEFT JOIN stock_transactions st ON st.product_id = p.id
  WHERE p.is_active = true
  GROUP BY p.id, p.name, p.sku, p.reorder_threshold;

09 Live Deployment Guide

1. Supabase Project Setup

Create a new project at supabase.com → copy SUPABASE_URL, SUPABASE_ANON_KEY, and SUPABASE_SERVICE_ROLE_KEY. Run schema migrations using Supabase CLI. 
# Install Supabase CLI
npm install -g supabase

# Initialize and link project
supabase init
supabase link --project-ref YOUR_PROJECT_REF

# Push schema to remote
supabase db push

2. Environment Variables (Vercel)

NEXT_PUBLIC_SUPABASE_URL=https://yourproject.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...   # server-only, never expose to client
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
RESEND_API_KEY=re_...
NEXT_PUBLIC_APP_URL=https://your-app.vercel.app

3. Deploy to Vercel

# Push to GitHub, then connect repo in Vercel dashboard
# Or via CLI:
npm install -g vercel
vercel --prod
After deployment, register your Stripe webhook endpoint at dashboard.stripe.com/webhooks pointing to https://your-app.vercel.app/api/webhooks/stripe. Enable events: payment_intent.succeeded, payment_intent.payment_failed, checkout.session.completed.
